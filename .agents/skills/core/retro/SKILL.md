# Retro — 振り返り

> プロジェクトの学びを構造化し、次回に活かす。

## 起動条件

- リリース・マイルストーン完了後
- 「振り返りをしよう」「レトロ」
- 失敗・インシデント解決後

## Steps

1. **事実確認**: 何を作ったか、何が起きたか
2. **Good / Bad / Try**: よかったこと、改善点、次回試すこと
3. **デザイン知見の抽出**: スタイル選択、カラー、アニメーションの学び
4. **LEARNINGS.md 更新**: 永続する知見を記録
5. **プロセス改善**: フレームワーク自体の改善提案

## Output

- 振り返りノート in `docs/retro-reports/retro-YYYY-MM-DD.md`
- `docs/LEARNINGS.md` 更新

## Do Not

- 抽象的すぎる学び（「もっと頑張る」→ 具体的に何を変えるか）
